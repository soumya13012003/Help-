# Help+ XAI Engine: Edge Case Handling Matrix

The Explainable AI subsystem is designed to remain highly available and robust under clinical production loads. The following matrix details how specific mathematical and computational edge cases are handled automatically by the dual-engine orchestrator.

| Edge Case / Anomaly | Impact on Clinical XAI | Orchestrator Mitigation Strategy | Status / Notification |
| :--- | :--- | :--- | :--- |
| **Collinearity (Highly Correlated Features)** | SHAP distributes credit equally among correlated features (e.g., `sys_bp` and `dia_bp`), potentially diluting the true primary driver in the UI. | Feature Grouping: Tabular features are pre-clustered logically (e.g., "Blood Pressure Vector") before visualization mapping to avoid confusing the clinician. | ⚪ Transparently handled in `visualization.py`. |
| **High-Dimensionality Text Spaces** | LIME surrogates can explode in computation time ($O(N^2)$) if clinical notes contain thousands of unique rare tokens, missing the latency budget. | LIME is hard-capped to generate perturbations using a maximum of `num_samples=500`. Text is pre-filtered to remove standard clinical stop-words before tokenization. | 🟡 Handled, but may slightly reduce LIME fidelity on massive texts. |
| **Computation Timeouts (>150ms)** | SHAP/LIME computation takes too long, delaying the critical path of the Clinician BFF returning the prediction payload. | **Asynchronous Fallback**: The core prediction is returned instantly. Explainability payloads are dispatched to a Celery/RabbitMQ background worker and pushed to the BFF via WebSockets when ready. | 🔴 Logged as an SLA violation in Datadog/Splunk. |
| **Near-Zero Confidence Differentials** | Prediction probability is exactly at the decision boundary (e.g., `0.500`). The sum of SHAP values cancels out completely. | The Natural Language Generator (NLG) defaults to a specific fallback declarative sentence: *"No specific primary drivers could be isolated with high confidence."* | ⚪ Transparently handled in `report_generator.py`. |
| **Missing Model Version in Background Summary** | The K-Means background dataset summary (`self.background_summary`) is out of sync with the newly deployed model version. | The `storage.py` layer explicitly tracks the `model_version_hash`. A drift alert is fired if the explanation model hash does not match the live inference model hash. | 🔴 Alert Data Eng team to trigger Airflow Retraining DAG. |
